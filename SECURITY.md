# Security Policy

## Supported Versions
We actively maintain security updates for the following versions:

| Version | Supported |
| ------- | ---------- |
| 5.1.x   | ✅         |
| 5.0.x   | ❌         |
| 4.0.x   | ✅         |
| < 4.0   | ❌         |

Only supported versions will receive security patches. Unsupported versions should be upgraded to ensure continued protection.

---

## Reporting a Vulnerability
We take security seriously and appreciate responsible disclosure.

### How to Report
Please email **quietopscb@gmail.com** with the subject line:  
`Security Vulnerability Report`

### What to Include
- A clear description of the vulnerability  
- Steps to reproduce (if applicable)  
- Potential impact and affected versions  
- Any suggested fixes or mitigations  

### Response Expectations
- Acknowledgment within **72 hours**  
- Status updates at least every **7 days** until resolution  
- If accepted, we will work on a fix and credit the reporter (if desired)  
- If declined, we will explain why (e.g., not reproducible, out of scope, or already fixed)  

### Confidentiality
Please do not disclose the vulnerability publicly until we have confirmed and released a fix. Coordinated disclosure helps protect all users.

---

## Commitment
Our goal is to maintain a secure and professional project environment.  
By reporting responsibly, you help strengthen the reliability and trustworthiness of this repository.
